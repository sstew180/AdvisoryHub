SKILL: How to Write a Project Charter
VERSION: 1.0
DOMAIN: Professional Practice
LEVEL: Base
JURISDICTION: Queensland

PURPOSE
A project charter is the foundational document that authorises a project to proceed. It defines what the project will deliver, why it exists, who is responsible, what it will cost, and when it will be complete. A signed project charter establishes the project manager's authority to use resources and make decisions within defined limits. Without a charter, a project lacks formal authorisation and clear scope -- the two most common causes of project failure.

WHEN TO USE THIS SKILL
- User asks to "write a project charter" or "prepare a project initiation document"
- User is starting a new project and needs to define scope and governance
- User needs to formally authorise a project before work begins
- User is preparing a project for submission to a steering committee or executive sponsor
- User asks to "document the project scope and governance" before mobilising resources

STRUCTURE
Project overview: Name, sponsor, project manager, date, and version.
Purpose and objectives: Why this project exists and what it will achieve. Specific and measurable.
Scope: What is included. What is explicitly excluded.
Deliverables: The specific outputs the project will produce.
Milestones: Key dates and decision points.
Budget: Total approved budget, funding source, and approval authority.
Governance: Project sponsor, steering committee, project manager, and key stakeholders.
Risks and assumptions: Key risks to delivery and the assumptions the charter is based on.
Constraints: Fixed deadlines, regulatory requirements, or resource limitations.
Authorisation: Signatures of the project sponsor and project manager.

KEY PRINCIPLES
- Scope must be unambiguous. The most expensive words in project management are "I thought that was included." Every significant item should be explicitly in scope or explicitly out of scope.
- Objectives must be measurable. "Improve the customer experience" is not an objective. "Reduce average call wait time from 8 minutes to 3 minutes by 31 December 2026" is an objective.
- The charter must be signed. An unsigned charter is a draft. Without the sponsor's signature, the project manager has no formal authority.
- Assumptions must be explicit. If the charter assumes a budget of $500,000 and the actual budget is $350,000, the charter will not deliver what it promises. State every significant assumption.
- Keep it concise. A project charter is an authorisation document, not a project plan. Two to four pages for most projects. A comprehensive project plan is a separate document.

QUALITY STANDARDS
A good project charter is one that a new steering committee member can read in ten minutes and know exactly what the project is, what it will cost, when it will be done, and who is accountable. If the scope is ambiguous enough that two people reading it would draw the line in different places, it needs to be tightened.

QUEENSLAND CONTEXT
Project charters for Queensland local government should align with the council's project management framework and any applicable gateway review requirements. Projects above the significant procurement threshold may require a Significant Contracting Plan (LGR s221) in addition to the project charter. Capital works projects should reference the approved budget allocation and the relevant program in the Long-Term Financial Forecast. Projects involving land, development, or infrastructure may require additional regulatory approvals that should be listed as constraints or dependencies. The project sponsor must hold the relevant financial delegation for the approved budget amount under LGA s257.

EXAMPLE LANGUAGE
Purpose and objectives: This project will replace the council's legacy fleet management system (FMS) with an integrated cloud-based platform. Objectives: (1) eliminate manual data entry for fleet transactions -- currently 2,300 transactions per month requiring 18 staff hours weekly; (2) achieve real-time integration with the council's Finance One financial system by 30 June 2027; (3) reduce fleet maintenance costs by 8% through improved scheduling within 12 months of go-live.
Scope exclusions: This project does not include: replacement of the council's fuel management system; changes to fleet procurement processes; replacement of the council's plant and equipment asset registers; or changes to the workshop management system.
Assumption: This charter assumes the Finance One integration can be achieved using the existing API without custom development. If custom development is required, the project manager will seek a budget variation approval before proceeding.
